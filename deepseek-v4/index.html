TL;DR

DeepSeek-V4 的目标是让百万 token 上下文成为常规能力。它用 CSA/HCA 混合注意力压缩 KV、mHC 稳定深层残差、Muon 加速训练，使 Pro 在 1M token 下仅需 V3.2 的 27% FLOPs 和 10% KV cache。

论文概述

问题：推理模型的 test-time scaling 越来越依赖长思考、长工具链和长文档，但 vanilla attention 的二次复杂度让百万 token 上下文的计算和 KV cache 成本难以承受。

方案：保留 DeepSeekMoE 和 MTP 主干，引入 CSA/HCA 混合注意力、mHC 残差连接、Muon 优化器，并配套训练、推理、KV cache、kernel 和后训练基础设施。

贡献：

  1. 提供 DeepSeek-V4-Pro 和 DeepSeek-V4-Flash 两个 MoE 语言模型：Pro 为 1.6T 总参数、49B 激活参数；Flash 为 284B 总参数、13B 激活参数，均支持 1M token 上下文。
  2. 设计 CSA/HCA 混合注意力，把 KV cache 沿序列维压缩，再结合稀疏选择、重压缩、滑动窗口和共享 KV MQA，显著降低长上下文推理成本。
  3. 将 mHC、Muon、FP4 QAT、异构 KV cache 管理、MoE 通信计算融合等系统工程组合起来，使百万 token 不只是单点实验，而是面向服务的设计。

图示内容：左侧展示 DeepSeek-V4-Pro-Max 与若干闭源/开源模型在知识、推理、Agent 指标上的对比；右侧展示 1M token 区间内单 token 推理 FLOPs 和累计 KV cache 的下降。 关键信息：在 1M token 上下文下，DeepSeek-V4-Pro 相比 DeepSeek-V3.2 只需要约 27% 单 token FLOPs 和 10% KV cache；DeepSeek-V4-Flash 进一步降到约 10% FLOPs 和 7% KV cache。

背景与动机

推理模型带来了新的 scaling 路径：不是只在训练阶段堆参数和数据，而是在推理阶段投入更多 token、工具调用、搜索和自验证。问题在于，这条路很快撞上 attention 的成本墙。

标准自注意力在长度为 n 的上下文中需要处理 n x n 的注意力关系。即使只看生成阶段，随着历史上下文增长，每生成一个 token 都要查询越来越长的 KV cache；当上下文来到 1M token，FLOPs、显存、cache 调度和服务延迟都会成为硬瓶颈。长文档问答、复杂 Agent、跨仓库代码修改、形式化证明这类任务还会把问题放大：它们既需要全局检索，又不能丢掉局部细节。

DeepSeek-V4 的设计可以理解为三层回答：

  1. 模型结构层：用 CSA/HCA 代替普通长上下文 attention，用 mHC 增强残差路径，用 DeepSeekMoE 保持参数效率。
  2. 训练稳定层：用 Muon、Anticipatory Routing、SwiGLU Clamping 和细粒度 checkpointing 处理 trillion-scale MoE 的训练不稳定。
  3. 服务工程层：重新设计 KV cache、落盘前缀复用、MoE 通信计算重叠和 batch-invariant/deterministic kernels，让长上下文模型可部署。

核心方法

整体架构

数据流可以简化为：

Input Tokens
  -> Embedding
  -> Transformer Block x L
       -> mHC Pre-Block Mixing
       -> CSA / HCA attention
       -> mHC Post-Block/Residual Mixing
       -> DeepSeekMoE
       -> mHC Post-Block/Residual Mixing
  -> Prediction Head
  -> MTP Modules

V4 不是完全重写 Transformer。它继承了 DeepSeek-V3 的两个核心组件：

  • DeepSeekMoE：仍采用共享专家与细粒度路由专家。V4 将计算 affinity score 的激活从 Sigmoid(.) 改为 Sqrt(Softplus(.))，继续使用 auxiliary-loss-free load balancing，并增加轻量 sequence-wise balance loss，防止单条序列内部极端不均衡。
  • MTP：Multi-Token Prediction 模块与目标保持和 V3 一致，MTP depth 设置为 1。

真正变化集中在残差、注意力和优化器。

mHC：把残差流扩宽，但不让信号失控

普通 Transformer 的残差流通常是一条 d 维向量。Hyper-Connections 的想法是把残差流扩成 n_hc x d，为层间信息传播增加一条新的 scaling 轴。设第 l 层前的残差状态为：

$$ X_l = [x_{l,1}; \ldots; x_{l,n_{hc}}]^T \in \mathbb{R}^{n_{hc} \times d} $$

标准 HC 用三个映射控制这一组残差状态：

$$ X_{l+1}=B_l X_l + C_l F_l(A_l X_l) $$

其中：

  • A_l in R^{1 x n_hc}：把扩展残差混合成实际层输入；
  • F_l：第 l 层主体，可以是 attention 或 MoE；
  • C_l in R^{n_hc x 1}：把层输出写回扩展残差；
  • B_l in R^{n_hc x n_hc}：残差状态之间的传播矩阵。

问题在 B_l。如果多层堆叠时 B_l 不受约束，残差信号可能在前向和反向传播中放大，造成数值不稳定。mHC 的做法是把 B_l 约束在 doubly stochastic matrices 的流形，也就是 Birkhoff polytope：

$$ B_l \in \mathcal{M} = \{M \in \mathbb{R}^{n \times n} \mid M\mathbf{1}_n=\mathbf{1}_n,\ \mathbf{1}_n^T M=\mathbf{1}_n^T,\ M \ge 0\} $$

这个约束带来两个性质：

  • ||B_l||_2 <= 1，残差传播是 non-expansive，不会系统性放大信号；
  • doubly stochastic matrices 对矩阵乘法封闭，深层堆叠时稳定性不会被层数轻易破坏。

mHC 的三个映射不是固定参数，而是由输入动态生成。先将 X_l 展平并 RMSNorm：

$$ \hat{X}_l=\mathrm{RMSNorm}(\mathrm{vec}(X_l)) $$

再生成未约束参数：

$$ \tilde{A}_l = \alpha_l^{pre} \cdot (\hat{X}_l W_l^{pre}) + S_l^{pre} $$
$$ \tilde{B}_l = \alpha_l^{res} \cdot \mathrm{Mat}(\hat{X}_l W_l^{res}) + S_l^{res} $$
$$ \tilde{C}_l = \alpha_l^{post} \cdot (\hat{X}_l W_l^{post})^T + S_l^{post} $$

其中 S 是静态偏置，alpha 是初始化为小值的门控因子。随后：

$$ A_l=\sigma(\tilde{A}_l), \quad C_l=2\sigma(\tilde{C}_l) $$

B_l 通过 Sinkhorn-Knopp 投影得到：

$$ M^{(0)}=\exp(\tilde{B}_l), \quad M^{(t)}=T_r(T_c(M^{(t-1)})) $$

论文中 t_max=20。直觉上，mHC 是在“不改变每层内部 hidden size”的前提下增加层间状态容量；Birkhoff 约束则相当于给这条更宽的残差高速路加上速度限制。

CSA：先压缩，再稀疏检索

CSA，全称 Compressed Sparse Attention，处理长上下文的路线是两步：

  1. 每 m 个 token 的 KV 被压成一个 compressed KV entry；
  2. 每个 query 只从压缩后的 KV entries 中选 top-k，再做核心 attention。

设输入 hidden states 为：

$$ H \in \mathbb{R}^{n \times d} $$

CSA 先生成两组 KV 候选和对应压缩权重：

$$ C^a = H W_a^{KV}, \quad C^b = H W_b^{KV} $$
$$ Z^a = H W_a^Z, \quad Z^b = H W_b^Z $$

随后在窗口内用权重和可学习位置偏置做 softmax：

$$ [S^a; S^b] = \mathrm{Softmax}_{row}([Z^a+B^a; Z^b+B^b]) $$

压缩 entry 由加权求和得到：

$$ C_i^{Comp} =\sum_j S_j^a \odot C_j^a +\sum_j S_j^b \odot C_j^b $$

论文中的 C^a/C^b 带有重叠设计，所以每个压缩 entry 实际吸收相邻区域的信息，但总体序列长度压缩到 1/m。

得到 C^{Comp} 后，CSA 使用 Lightning Indexer 做稀疏选择。query token t 的低秩 indexer query 为：

$$ c_t^Q = h_t W^{DQ} $$
$$ [q^I_{t,1}; \ldots; q^I_{t,n_h^I}] = c_t^Q W^{IUQ} $$

索引分数为：

$$ I_{t,s} = \sum_{h=1}^{n_h^I} w^I_{t,h} \cdot \mathrm{ReLU}(q^I_{t,h} \cdot K_s^{IComp}) $$

然后选出：

$$ C_t^{SprsComp} = \{C_s^{Comp} \mid I_{t,s} \in \mathrm{Top}\text{-}k(I_{t,:})\} $$

最后在被选中的压缩 KV 上做共享 KV 的 MQA：

$$ [q_{t,1}; \ldots; q_{t,n_h}] = c_t^Q W^{UQ} $$
$$ o_{t,i} = \mathrm{CoreAttn}( \mathrm{query}=q_{t,i}, \mathrm{key}=C_t^{SprsComp}, \mathrm{value}=C_t^{SprsComp}) $$

CSA 还额外拼接一小段未压缩的 sliding window KV entries。原因很直接：严格因果压缩会让 query 看不到同一个压缩 block 内的其他 token，而语言建模中最近 token 往往最有用。滑窗分支补回局部细节。

HCA：更重压缩，但保留 dense attention

HCA，全称 Heavily Compressed Attention，可以看成 CSA 的互补分支。它不做 top-k 稀疏选择，而是把压缩率拉得更高：

$$ C = H W^{KV}, \quad Z = H W^Z $$
$$ S_{m'i:m'(i+1)-1} = \mathrm{Softmax}_{row}(Z_{m'i:m'(i+1)-1}+B) $$
$$ C_i^{Comp} = \sum_{j=m'i}^{m'(i+1)-1} S_j \odot C_j $$

HCA 将序列长度压到 1/m'，其中 m' >> m。在 DeepSeek-V4 配置中，CSA 的 m=4，HCA 的 m'=128。这意味着：

  • CSA 更适合保留相对细粒度的全局信息，但要靠 top-k 控制计算；
  • HCA 信息更粗，但 dense attention 避免了稀疏检索漏召的问题；
  • 两者 interleaved 使用，让模型在不同层交替获得“较细的稀疏全局视野”和“很粗但全覆盖的全局视野”。

CSA 和 HCA 共享几项细节：

  • RMSNorm on queries and KV entries：避免 attention logits 爆炸；
  • Partial RoPE：只在 query、KV entry 的最后 64 维使用 RoPE；
  • 输出端反向 RoPE：由于 compressed KV 同时作为 key/value，attention 输出会携带绝对位置信息，论文对输出最后 64 维施加位置 -i 的 RoPE，让输出更像相对位置表征；
  • Attention Sink：给每个 head 一个可学习 sink logit，使注意力总分可以小于 1，甚至接近 0；
  • Grouped Output Projection：先按组投影 attention head 输出，再合并到 hidden size，降低大 c n_h 直接投影的成本。

一个简化版伪代码如下：

def csa_layer(h):
    ca, cb = h @ W_kv_a, h @ W_kv_b
    za, zb = h @ W_z_a, h @ W_z_b
    c_comp = overlapping_weighted_compress(ca, cb, za, zb, block=m)

    indexer_keys = compress_for_indexer(h, block=m)
    q_latent = h @ W_dq
    indexer_queries = q_latent @ W_iuq
    scores = lightning_indexer(indexer_queries, indexer_keys)
    selected = topk(c_comp, scores, k=top_k)

    local_kv = recent_uncompressed_kv(h, window=n_win)
    kv = concat(selected, local_kv)
    out = shared_kv_mqa(q_latent, kv)
    return grouped_output_projection(out)


def hca_layer(h):
    c = h @ W_kv
    z = h @ W_z
    c_comp = weighted_compress(c, z, block=m_prime)

    q_latent = h @ W_dq
    local_kv = recent_uncompressed_kv(h, window=n_win)
    kv = concat(c_comp, local_kv)
    out = shared_kv_mqa(q_latent, kv)
    return grouped_output_projection(out)

Muon：让大矩阵更新更接近正交方向

DeepSeek-V4 对多数模块使用 Muon，对 embedding、prediction head、mHC 的静态偏置和门控、RMSNorm 权重继续使用 AdamW。Muon 的更新可以概括为：

G_t = grad_W L_t(W_{t-1})
M_t = mu * M_{t-1} + G_t
O'_t = HybridNewtonSchulz(mu * M_t + G_t)
O_t = O'_t * sqrt(max(n, m)) * gamma
W_t = W_{t-1} * (1 - eta * lambda) - eta * O_t

Hybrid Newton-Schulz 用于把更新矩阵近似正交化。若 M = U Sigma V^T，目标是得到近似 U V^T。论文采用 10 次迭代，前 8 次用 (a,b,c)=(3.4445,-4.7750,2.0315) 快速逼近，后 2 次用 (2,-1.5,0.5) 稳住奇异值。

这里的设计取舍是：Muon 对矩阵参数更合适，但不必强行覆盖所有参数。小型标量/向量参数、归一化权重和 mHC 门控更像 element-wise 参数，用 AdamW 更稳、更直接。

训练和推理基础设施

MoE 的专家并行通常卡在 dispatch/combine 通信。DeepSeek-V4 把一个 MoE 层拆成通信受限的 Dispatch/Combine 和计算受限的 Linear-1/Linear-2，再按 expert wave 调度。稳态下，当前 wave 的计算、下一 wave 的 token transfer、已完成 expert 的结果发送可以并发进行。论文报告相对强 non-fused baseline，一般推理 workload 有 1.50-1.73x 加速，RL rollout 和高速 Agent serving 等小 batch 场景最高 1.96x。

论文还给出一个硬件设计视角的条件。设峰值计算吞吐为 C，互联带宽为 B，当：

$$ \frac{C}{B} \le \frac{V_{comp}}{V_{comm}} $$

通信就可以被计算隐藏。对 DeepSeek-V4-Pro，该条件可化简到：

$$ \frac{C}{B} \le 2d = 6144\ \mathrm{FLOPs/Byte} $$

这句话的含义是：不是带宽越大越好，而是计算/通信比例达到隐藏通信的平衡点后，继续堆带宽收益会下降。

推理侧的 KV cache 被拆成两类：

  • State Cache：保存 SWA 最近窗口 KV，以及 CSA/HCA 尚未凑够压缩 block 的尾部 hidden states；
  • Classical KV Cache：保存已经压缩完成的 CSA/HCA KV blocks。

这套布局是为 CSA/HCA 的异构 cache 服务的：不同层可能有 CSA 或 HCA，压缩率不同，indexer 也有自己的 KV，SWA 又有独立的命中和淘汰策略。传统 PagedAttention 的“每层 cache 形态相同”假设不再成立。

共享前缀复用时，DeepSeek-V4 还支持 on-disk KV cache：

  • CSA/HCA：压缩 KV 直接落盘，命中前缀时读到最后一个完整压缩 block；
  • SWA：因为未压缩且每层都有，体积约为压缩 CSA/HCA 的 8 倍，论文讨论了 full caching、periodic checkpointing、zero caching 三种策略，用存储换重计算。

训练细节

模型配置

DeepSeek-V4-Flash：

  • 43 层，hidden size d=4096；
  • 前 2 层使用 pure sliding window attention，后续 CSA/HCA 交替；
  • CSA：m=4，indexer heads n_h^I=64，indexer head dim c^I=128，top-k 512；
  • HCA：m'=128；
  • CSA/HCA：query heads n_h=64，head dim c=512，query compression dim d_c=1024，output groups g=8，每组中间维度 d_g=1024；
  • sliding window size n_win=128；
  • MoE：每层 1 个 shared expert、256 个 routed experts，每 token 激活 6 个 routed experts，expert intermediate hidden dim 为 2048；
  • mHC：n_hc=4，Sinkhorn-Knopp 迭代 t_max=20；
  • 总参数 284B，激活参数 13B。

DeepSeek-V4-Pro：

  • 61 层，hidden size d=7168；
  • 前 2 层使用 HCA，后续 CSA/HCA 交替；
  • CSA：m=4，top-k 1024；
  • HCA：m'=128；
  • query heads n_h=128，head dim c=512，query compression dim d_c=1536，output groups g=16；
  • MoE：每层 1 个 shared expert、384 个 routed experts，每 token 激活 6 个 routed experts，expert intermediate hidden dim 为 3072；
  • mHC：n_hc=4，t_max=20；
  • 总参数 1.6T，激活参数 49B。

预训练

训练语料超过 32T tokens，包含数学、代码、网页、长文档、多语言和 agentic data。tokenizer 延续 DeepSeek-V3，并加入少量上下文构造 special tokens，词表保持 128K；FIM 和 token-splitting 策略也继承自 V3。V4 额外使用 sample-level attention masking。

训练设置：

  • Flash 训练 32T tokens，Pro 训练 33T tokens；
  • batch size 按 token 数逐步增大，Flash 最大 75.5M，Pro 最大 94.4M；
  • Flash peak LR 2.7e-4，最终 2.7e-5；Pro peak LR 2.0e-4，最终 2.0e-5；
  • 序列长度从 4K 逐步扩到 16K、64K、1M；
  • Flash 前 1T tokens 使用 dense attention warmup，之后在 64K 长度阶段引入 sparse attention；Pro 的 dense attention 阶段更长；
  • CSA 的 lightning indexer 先单独 warmup，再进入主训练；
  • balance bias update speed 为 0.001，sequence-wise balance loss weight 为 1e-4；
  • MTP loss weight 大部分训练为 0.3，学习率衰减开始后降到 0.1。

训练稳定性

论文坦诚提到 trillion-parameter MoE 训练中遇到 loss spikes，且 spike 与 MoE 层 outliers、路由机制有关。两个工程技巧很关键：

Anticipatory Routing：第 t 步用当前参数 theta_t 计算 backbone features，但路由 indices 使用历史参数 theta_{t-Delta t}。实现上提前在 t-Delta t 步读取数据并缓存未来会用到的路由。完整启用时约有 20% wall-clock overhead，但论文采用自动触发机制：检测到 loss spike 后短回滚并启用一段时间，稳定后恢复标准训练，因此总体额外开销很小。

SwiGLU Clamping：将 SwiGLU 的 linear component clamp 到 [-10, 10]，gate component 上界设为 10，用于抑制 outliers。

后训练

后训练不是一个单一模型直接混合 RL，而是两阶段：

  1. 先训练多个 domain specialists：数学、代码、Agent、指令跟随等领域分别经过 SFT 和 GRPO；
  2. 再用 multi-teacher On-Policy Distillation 把多个专家合并到一个统一模型。

OPD 目标为：

$$ \mathcal{L}_{OPD}(\theta) = \sum_{i=1}^{N} w_i \cdot D_{KL}(\pi_\theta \| \pi_{E_i}) $$

其中 pi_theta 是 student，pi_{E_i} 是第 i 个 expert teacher。这是 reverse KL，并且轨迹来自 student 自己，所以是 on-policy。论文没有采用常见的 token-level KL 近似，而是使用 full-vocabulary logit distillation，以降低梯度估计方差、提升蒸馏稳定性。为了让这个方案能跑起来，系统只缓存 teacher 的 last-layer hidden states，需要时再通过对应 prediction head 重建 full logits；每个 mini-batch 尽量只加载一个 teacher head，减少显存压力。

后训练还加入 FP4 QAT：

  • MoE expert weights 使用 FP4(MXFP4)；
  • CSA indexer 的 QK path 使用 FP4；
  • index scores 从 FP32 量化到 BF16，使 top-k selector 获得 2x speedup，同时保持 99.7% KV entry recall。

实验分析

Base model：结构效率确实转化成了能力

Table 1 对比 DeepSeek-V3.2-Base、DeepSeek-V4-Flash-Base、DeepSeek-V4-Pro-Base。最有信息量的不是“Pro 更大所以更强”，而是 Flash 的参数效率。

DeepSeek-V4-Flash-Base 只有 13B activated params，而 DeepSeek-V3.2-Base 是 37B activated params；但 Flash 在许多 benchmark 上超过 V3.2-Base，例如：

  • MMLU-Pro：68.3 vs 65.5；
  • FACTS Parametric：33.9 vs 27.1；
  • LongBench-V2：44.7 vs 40.2；

深度理解问答

Q1：为什么要同时使用 CSA 和 HCA，而不是只用一种 attention？

CSA 和 HCA 分别解决不同的失败模式。

CSA 的压缩率较低，m=4，再通过 top-k sparse selection 找相关 compressed KV entries。它更适合保留细粒度全局信息，但 top-k selector 可能漏掉某些长期依赖。

HCA 的压缩率很高，m'=128，但不做稀疏选择，而是在重压缩后的序列上 dense attention。它的信息更粗，却不依赖 top-k 召回。把两者交替使用，相当于在层间混合“细粒度但稀疏的全局检索”和“粗粒度但全覆盖的全局摘要”。滑动窗口分支再负责最近 token 的局部细节。

Q2：mHC 为什么要把 B_l 约束成 doubly stochastic matrix？

扩展残差流会增加表达能力，但也增加信号传播不稳定的风险。B_l 是残差状态在层间传播的矩阵，如果它的谱范数持续大于 1，多层堆叠后前向激活和反向梯度都可能放大。

doubly stochastic matrix 的行和列都归一，且非负。论文利用它的两个性质：||B_l||_2 <= 1，以及这类矩阵对乘法封闭。前者限制单层扩张，后者保证多层堆叠后仍不会轻易破坏稳定性。Sinkhorn-Knopp 投影则提供了一个可训练的实现路径。

Q3：为什么 CSA/HCA 还需要 sliding window 分支？

压缩 attention 严格保证因果性时，query 只能看前面的完整压缩块。若 query 位于某个 block 内，它无法访问同 block 中尚未压缩完成的近邻 token。语言建模里，最近 token 对下一个 token 的预测通常非常重要，例如括号闭合、变量名延续、短程指代。

滑动窗口分支保留最近 n_win=128 个未压缩 KV entries，用少量 cache 换回局部精度。可以把它理解成：CSA/HCA 负责远处地图，SWA 负责脚下路面。

Q4：Attention Sink 在这里起什么作用？

普通 softmax attention 会把权重归一到 1，意味着每个 query 必须把全部注意力分配给某些 token。长上下文中，有些 head 在某些位置可能不需要强行关注任何历史内容。

Attention Sink 给 denominator 额外加入一个可学习项：

$$ s_{h,i,j} = \frac{\exp(z_{h,i,j})} {\sum_k \exp(z_{h,i,k})+\exp(z'_h)} $$

这样注意力权重总和可以小于 1。当 sink logit 较大时，该 head 可以选择“少看历史”。这对压缩/稀疏 attention 特别有用，因为被选中的 KV 并不总是足够可靠。

Q5：Muon 为什么和 ZeRO 有冲突？

AdamW 这类 element-wise optimizer 可以把一个参数矩阵切给多个 rank，各自更新自己的 shard。Muon 需要完整梯度矩阵来做 Newton-Schulz 正交化，如果矩阵被普通 ZeRO 切碎，就无法直接计算完整矩阵更新。

DeepSeek-V4 的处理是 hybrid ZeRO bucket assignment：dense 参数限制 ZeRO 并行规模，用 knapsack 分配完整矩阵到 rank；MoE 参数则按 expert 的逻辑独立矩阵组织，避免切开单个矩阵。这个设计用一些冗余计算换取可控的内存和通信。

Q6：为什么 OPD 用 full-vocabulary logit distillation，而不是 token-level KL 近似？

token-level KL 近似更省资源，但梯度估计方差大，容易影响训练稳定性。DeepSeek-V4 的目标是把十多个 specialist teacher 合并到一个统一模型，如果蒸馏信号本身噪声过大，很容易出现某些能力被覆盖或互相干扰。

full-vocabulary logit distillation 保留 teacher 对整个词表的分布信息，能更细地表达“哪些替代 token 也合理、哪些明显不合理”。代价是显存和 I/O 极高，所以论文缓存 teacher last-layer hidden states，需要时再重建 logits，并用 teacher head 调度和 TileLang kernel 降低开销。

Q7：DeepSeek-V4 的百万 token 能力有什么边界？

第一，质量仍会随长度退化。MRCR 在 128K 后已经能看到下降，只是 1M 处仍保持可用。 第二，架构复杂度高。CSA、HCA、SWA、mHC、Muon、QAT、异构 cache、落盘 cache、确定性 kernel 都需要协同，复现难度远高于单一 attention trick。 第三，训练稳定性的机理还不完全清楚。论文明确承认 Anticipatory Routing 和 SwiGLU Clamping 有效，但理论解释仍不足。 第四，长上下文 Agent 的收益依赖真实工具调用路径。如果框架把工具结果模拟成普通 user message，论文提到部分 interleaved thinking 管理策略可能无法触发。

总结

读完后应抓住的三点

  • DeepSeek-V4 的百万 token 能力不是靠单个 attention trick，而是“混合注意力 + 残差稳定 + 优化器 + KV cache + kernel + 后训练系统”的组合工程。
  • CSA/HCA 是架构主轴：CSA 用较低压缩率和 top-k 稀疏选择保留细粒度全局信息，HCA 用高压缩率和 dense attention 提供低成本全覆盖信息。
  • Pro 和 Flash 的定位清晰：Pro 追求能力上限，Flash 追求成本效率；Flash 的 base model 已经展示出较强参数效率，但复杂代码/数学任务仍有短板。

局限性

  • 架构明显复杂，组件间耦合强，后续可能需要更原则化的简化。
  • Anticipatory Routing、SwiGLU Clamping 等稳定性技巧尚缺系统理论解释。
  • 1M 上下文质量并非无损，超长检索和真实长文档任务仍会随长度下降。
  • 部分真实任务和内部评测依赖 DeepSeek 自建框架、数据和人工评价，外部复验成本较高。

适用场景

  • 长文档、多文档、代码仓库、长工具链 Agent 等需要 128K 到 1M 上下文的任务；
  • 有较强推理预算管理需求，希望在 High/Max 模式下获得 test-time scaling 的场景；
  • 可接受复杂 serving stack，并能投入 KV cache 管理、低精度 kernel、落盘缓存和 sandbox 基础设施的团队。
